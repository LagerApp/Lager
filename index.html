What is Lager?

Lager is a logging service that provides a mobile web interface for your devices.

It runs on your server and sets up a authenticable web interface where you can add servers and service logs to monitor.

Supported platforms

  • OS X

  • Ubuntu, Debian and other Debian-based distributions

Dependencies

  • Ruby 2.2

  • SQLite3

How it works

  1. Clone LagerApp onto the server you want Sinatra to run on.

  2. Install ruby-2.2.2 and sqlite3.

  3. Run bundle install to install the gem dependencies. Run gem install bundler if you do not have Bundler installed.

  4. Run setup.sh (link to script). This will set up the web app and a sqlite database.

  5. Run ruby app.rb to start the server. We recommend using Supervisor to run the server in the background.

  6. Go to /init on the server to create the initial user with username: admin and password: admin. You can also supply username and password as optional query parameters (example: /init?username=newusername&password=newpassword).

  7. Go to /init_script on the server to generate another script to run on the servers you want to monitor. The generated script makes use of a private/public key pair and a restricted unix account that only has access to log folders on other servers. The script template can be found here.

  8. Visit the domain/IP address of the server at port 4000 to access the web interface. Log in with the username and password that you chose above.
